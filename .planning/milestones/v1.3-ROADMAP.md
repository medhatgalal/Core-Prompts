# Milestone v1.3: Controlled Extension Re-entry

**Status:** ✅ SHIPPED 2026-03-11
**Phases:** 08-10
**Total Plans:** 9

## Overview

v1.3 re-opened the previously deferred extension surface in a controlled way. It added shared fail-closed extension contracts, deterministic URL ingestion through immutable snapshots, and a simulate-first controlled execution plane with explicit approvals, closed registry mapping, deterministic journaling, and hermetic adapter behavior.

## Milestone Summary

**Key Decisions:**
- Re-enter extension scope only behind explicit versioned policy contracts and deterministic rule IDs.
- End the URL network boundary at immutable content-addressed snapshots before sanitization.
- Keep execution simulate-first by default and require exact approval + static registry alignment for any execute-approved path.

**Issues Resolved:**
- Closed deferred `EXT-01` and `EXT-02` scope without violating local-only baseline guarantees.
- Added real public URL validation for both accepted and rejected sources.
- Added a reusable one-URL end-to-end runner for operator understanding and release smoke testing.

**Issues Deferred:**
- Authenticated retrieval profiles remain deferred to a future milestone.
- Multi-document crawling and transactional multi-tool execution remain deferred.

**Technical Debt Incurred:**
- None recorded in milestone audit.
